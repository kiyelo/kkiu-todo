# 끼우 테스트 세션 가이드

이 문서는 사용자가 앱을 직접 눌러보면서 Codex에 수정 지시를 전달하기 위한 기준입니다. 과거 QA 결과는 `qa/`에 보관되어 있지만, 현재는 자동 재실행 테스트가 아니라 참고 기록입니다.

## 테스트 시작

Codex가 개발 서버를 실행한 뒤 다음 주소를 엽니다.

- 이 컴퓨터: `http://localhost:5173/kkiu-todo/`
- 같은 Wi-Fi의 휴대폰: 개발 서버가 표시하는 Network 주소의 `/kkiu-todo/`

`.env.local`에 Supabase 값이 있으면 로그인·끼리 동기화 모드로 실행됩니다. 실제 데이터와 섞이지 않도록 테스트 전용 계정을 사용합니다. Supabase 값이 없으면 브라우저 로컬 저장 모드로 실행됩니다.

## 지시하는 형식

긴 설명은 필요 없습니다. 아래 네 가지만 알려주면 Codex가 화면과 코드를 찾아 수정합니다.

```text
화면: 끼리 > 할 일 입력
행동: 담당자 2명을 선택하고 함께 완료로 추가
문제: 새로고침하면 담당자가 1명만 남음
원하는 모습: 선택한 담당자가 모두 유지됨
```

가능하면 기기 종류와 세로/가로 방향도 덧붙입니다. 스크린샷이 있으면 가장 빠르게 재현할 수 있습니다.

## 첫 번째 통합 테스트 순서

각 항목에서 이상이 보이면 다음 항목으로 넘어가지 않고 바로 지시해도 됩니다.

1. 시작·계정
   - 첫 화면, 로그인, 가입 확인 메일, 비밀번호 재설정
   - 로그아웃 후 다시 로그인했을 때 데이터 유지
2. 개인 할 일
   - 원하는 순번에 추가, 수정, 완료, 복원, 삭제
   - 세로 드래그·빠른 스와이프·키보드 입력
3. 검색·완료 목록
   - 긴 제목 검색, 결과 이동, 오늘/이전 그룹, 개별 복원
4. 끼리
   - 생성, 초대 코드 복사, 다른 계정 참여, 나가기
   - 프로필 변경, 멤버 순서, 읽지 않음 표시
5. 담당자
   - 한 명 배정
   - 여러 명을 선택한 “함께 완료”와 “각자 완료”
   - 저장 후 새로고침 및 다른 기기에서 동일하게 보이는지
6. 더보기
   - 언어 전환, 백업·복원, 초기화, 약관·개인정보·문의
7. 작은 화면
   - 320px 너비와 Android 시스템 글자 크기 확대 상태
   - 키보드가 열린 상태에서 입력창·저장 버튼이 가려지지 않는지

## Android 전용 테스트가 시작되면

- 뒤로가기 버튼이 모달 → 화면 → 앱 종료 순서로 자연스럽게 동작하는지
- 상태 표시줄·내비게이션 바·노치 영역과 UI가 겹치지 않는지
- 앱을 백그라운드에 두었다가 돌아왔을 때 로그인과 작성 내용이 유지되는지
- 이메일 인증 링크와 `?join=CODE` 초대 링크가 앱으로 돌아오는지
- 오프라인/불안정한 네트워크에서 데이터 손실 없이 오류를 안내하는지

## 현재 우선 확인할 위험

- 여러 담당자를 선택하면 담당자별 단일 `assignee_id` 할 일이 각각 생성되는지, 재접속 후에도 개수·순서·담당자가 유지되는지 확인해야 합니다.
- 계정을 만들 수 있으므로 출시 전 인앱 계정 삭제와 외부 삭제 요청 페이지가 모두 필요합니다.
- 테스트 결과 JSON은 자동화 스크립트가 아니므로 회귀 테스트 체계를 별도로 만들어야 합니다.
